# Plan: Salvare Tema Grafică Completă în Folder Portabil

## Descriere

Crearea unui folder dedicat care conține toate fișierele necesare pentru tema grafică completă a aplicației, organizate într-un format portabil și reutilizabil în alte aplicații. Folderul va include tokeni de design, configurații de temă, stiluri CSS, și documentație completă pentru integrare.

## Context Actual

### Fișiere Portabile Existente

Există deja fișiere portabile la rădăcina proiectului:
- `design-system-theme.json` - Tema completă în format JSON
- `design-system-theme.ts` - Tema completă în format TypeScript cu tipizare
- `design-system-theme.schema.json` - Schema JSON pentru validare
- `design-system-theme.package.json` - Configurație npm package
- `DESIGN_SYSTEM_THEME_README.md` - Documentație utilizare

### Structura Design System Existente

- `src/design-system/tokens/` - Tokeni individuali (colors, typography, spacing, shadows, borders, transitions, breakpoints, z-index)
- `src/design-system/themes/` - Configurații teme (light, dark)
- `src/design-system/styles/` - Fișiere CSS (base.css, components.css, utilities.css, themes.css)
- `src/design-system/components/` - Componente React (Button, Input, Card, Badge, Modal, Tooltip, Table, Form)
- `src/design-system/hooks/` - Hooks (useTheme, useMediaQuery)
- `src/design-system/utilities/` - Utilitare (classNames)

### Probleme Identificate

1. **Fișierele portabile sunt dispersate** la rădăcina proiectului, nu într-un folder dedicat
2. **Lipsă organizare clară** pentru utilizare în alte aplicații
3. **Nu există un folder standalone** care să conțină tot ce este necesar pentru tema grafică
4. **Lipsă instrucțiuni clare** pentru copierea și utilizarea în alte proiecte

## Obiective

1. Crearea unui folder dedicat `theme/` sau `portable-theme/` care conține toate fișierele necesare
2. Organizarea fișierelor într-o structură logică și ușor de navigat
3. Includerea tuturor tokenilor, stilurilor CSS, și documentației necesare
4. Crearea unui README clar cu instrucțiuni de utilizare în alte aplicații
5. Păstrarea compatibilității cu structura existentă a proiectului

## Detalii Tehnice

### 1. Folder Principal (`theme/`)

**Locație:** La rădăcina proiectului, alături de `src/`, `backend/`, etc.

**Scop:** Folder standalone care poate fi copiat întreg în alte proiecte sau publicat ca npm package.

### 2. Tokeni (`theme/tokens/`)

#### 2.1. Fișiere de Copiat

**`theme.json`** (copiat din `design-system-theme.json`):
- Tema completă în format JSON
- Compatibil cu orice aplicație JavaScript/TypeScript
- Poate fi importat direct: `const theme = require('./theme.json')`

**`theme.ts`** (copiat din `design-system-theme.ts`):
- Tema completă în format TypeScript cu tipizare completă
- Export `theme` și `THEME_VERSION`
- Export tipuri TypeScript pentru type safety
- Poate fi importat: `import { theme, THEME_VERSION } from './theme'`

**`schema.json`** (copiat din `design-system-theme.schema.json`):
- Schema JSON pentru validare
- Poate fi folosit în IDE-uri pentru autocomplete
- Validare automată a structurii

#### 2.2. Documentație Tokeni (`theme/tokens/README.md`)

**Conținut:**
- Descriere fișiere disponibile

## Scripturi de Generare

### Script Generare CSS Variables (`scripts/generate-css-variables.js`)

**Scop:** Generare automată CSS variables din `theme.json`

**Implementare:**
```javascript
const fs = require('fs');
const theme = require('../theme/tokens/theme.json');

function generateCSSVariables(theme) {
  let css = ':root {\n';
  
  // Colors
  Object.entries(theme.colors.primary).forEach(([key, value]) => {
    css += `  --color-primary-${key}: ${value};\n`;
  });
  
  // ... generează pentru toate categoriile
  
  css += '}\n\n.dark {\n';
  // Dark theme overrides
  css += '}\n';
  
  return css;
}

const css = generateCSSVariables(theme);
fs.writeFileSync('theme/configs/css-variables.css', css);
```

**Utilizare:** `node scripts/generate-css-variables.js`

### Script Copiere Fișiere (`scripts/copy-theme-files.js`)

**Scop:** Automatizare copiere fișiere în folder `theme/`

**Implementare:**
```javascript
const fs = require('fs');

// Copiere fișiere existente

  { from: 'design-system-theme.schema.json', to: 'theme/tokens/schema.json' },

filesToCopy.forEach(({ from, to }) => {
  fs.copyFileSync(from, to);
  console.log(`Copied ${from} → ${to}`);
});
```

**Utilizare:** `node scripts/copy-theme-files.js`

## Algoritmi și Logică

### 1. Generare CSS Variables

**Algoritm:**
1. Citește `theme.json`
2. Parcurge recursiv toate proprietățile
3. Generează nume CSS variable în format kebab-case: `--color-primary-600`
4. Generează valori CSS
5. Scrie în `css-variables.css`

**Exemplu:**
```javascript
theme.colors.primary[600] → --color-primary-600: #4f46e5;
theme.spacing[4] → --spacing-4: 1rem;
theme.typography.fontSize.base[0] → --font-size-base: 1rem;
```

### 2. Validare Structură

**Algoritm:**
1. Validează `theme.json` folosind `schema.json`
2. Verifică consistență între `theme.json` și `theme.ts`
3. Verifică că toate fișierele CSS există
4. Verifică că versiunile sunt sincronizate

**Script:** `scripts/validate-theme.js` (există deja, poate fi extins)

## Considerații de Implementare

- Păstrare compatibilitate cu fișierele existente la rădăcina proiectului
